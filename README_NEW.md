# 網格地圖策略評估與價值迭代 (Grid Map Policy & Value Iteration)

## 項目概述 (Project Overview)

這個項目實現了一個完整的強化學習網格地圖應用，包含三個主要作業部分：

### HW1-1: 網格地圖開發 (Grid Map Development)
- 用戶可指定維度 n（5-9）的 n×n 網格
- 通過滑鼠點擊指定：
  - **起始單元格** (起點) - 顯示為綠色
  - **結束單元格** (目標) - 顯示為紅色  
  - **障礙物** - 顯示為灰色 (數量: n-2個)

### HW1-2: 隨機策略與策略評估 (Policy Evaluation)
- 生成隨機策略，為每個狀態(除了障礙物和目標)分配隨機行動(上下左右)
- 使用**策略評估算法**計算每個狀態的價值函數 V(s)
- 結果顯示：
  - **策略矩陣**：顯示每個狀態的行動 (↑ ↓ ← →)
  - **價值矩陣**：顯示每個狀態的期望回報 V(s)

### HW1-3: 價值迭代算法 (Value Iteration)
- 實現**價值迭代算法**以計算最優策略
- 通過反復更新狀態值來尋找最優行動
- 結果顯示：
  - **最優策略矩陣**：顯示每個狀態下的最優行動
  - **最優價值矩陣**：顯示在最優策略下的期望回報 V*(s)

## 應用參數 (Application Parameters)

| 參數 | 值 | 說明 |
|------|-----|------|
| 折扣因子 (γ) | 0.9 | 衡量未來獎勵的重要性 |
| 收斂閾值 (θ) | 1e-4 | 價值函數變化小於此值時停止迭代 |
| 步驟成本 | -1.0 | 每走一步的代價 |
| 目標獎勵 | 0.0 | 到達目標狀態的獎勵 |
| 最大迭代次數 | 1000 | 防止無限循環 |

## 算法說明 (Algorithm Descriptions)

### 策略評估算法 (Policy Evaluation Algorithm)

給定策略 π，計算該策略下的狀態價值函數 V(s)：

```
初始化: V(s) = 0 對所有狀態 s
重複直到收斂:
  Δ ← 0
  對每個狀態 s:
    v ← V(s)
    a ← π(s)  [根據策略選擇動作]
    s' ← 執行動作 a 後的下一狀態
    V(s) ← R(s,a) + γ·V(s')
    Δ ← max(Δ, |v - V(s)|)
  如果 Δ < θ: 跳出循環
```

**複雜度**: O(n² × 迭代次數) 其中 n 是網格大小

### 價值迭代算法 (Value Iteration Algorithm)

同時優化策略和價值函數，找到最優策略：

```
初始化: V(s) = 0 對所有狀態 s
重複直到收斂:
  Δ ← 0
  對每個狀態 s:
    v ← V(s)
    對每個可能的動作 a:
      計算 Q(s,a) = R(s,a) + γ·V(s')
    V(s) ← max_a Q(s,a)
    π(s) ← argmax_a Q(s,a)  [選擇使 Q 値最大的動作]
    Δ ← max(Δ, |v - V(s)|)
  如果 Δ < θ: 跳出循環
```

**複雜度**: O(n² × 動作數 × 迭代次數)

**優點**: 同時計算最優策略，通常比策略迭代快

## 輸出矩陣說明 (Matrix Output Description)

### 策略矩陣 (Policy Matrix)
顯示每個狀態下的最優或隨機動作：

| 符號 | 含義 |
|------|------|
| **↑** | 最優/隨機動作為向上 |
| **↓** | 最優/隨機動作為向下 |
| **←** | 最優/隨機動作為向左 |
| **→** | 最優/隨機動作為向右 |
| **X** | 障礙物（無法訪問的狀態） |
| *空白* | 目標狀態（終止狀態） |

### 價值矩陣 (Value Matrix)  
顯示每個狀態的期望長期回報：

| 內容 | 含義 |
|------|------|
| **数字** | 狀態值 V(s)，表示從該狀態開始的期望累積回報 |
| **0.0** | 目標狀態（終止狀態，回報為 0） |
| **X** | 障礙物（無法訪問的狀態） |
| **負值** | 表示需要付出成本達到目標（因為每步成本為 -1） |

**值的解釋示例**:
- V(s) = -5.0：從此狀態到目標的最短路徑約為 5 步
- V(s) = -10.0：從此狀態到目標的最短路徑約為 10 步
- V(s) 越接近 0：狀態越接近目標

## 動作定義 (Action Definitions)

應用中定義的四個基本動作：

```python
actions = [
    (-1, 0),  # 0: Up    (向上，行數減 1)
    (1, 0),   # 1: Down  (向下，行數加 1)
    (0, -1),  # 2: Left  (向左，列數減 1)
    (0, 1)    # 3: Right (向右，列數加 1)
]
```

**邊界處理**: 如果動作會導致越界或碰到障礙物，智能體停留在原位置

## 環境特性 (Environment Characteristics)

- **狀態空間**: {(r,c) | 0 ≤ r,c < n, (r,c) ∉ obstacles ∪ {end}}
- **動作空間**: {上, 下, 左, 右}
- **過渡模型**: 確定性（Deterministic）- 動作的結果是確定的
- **獎勵模型**: 
  - R(s,a) = -1 對所有非終止狀態
  - R(goal) = 0（終止狀態無獎勵）
- **邊界**: 環境受邊界和障礙物限制

## 常見問題 (FAQ)

**Q: 價值矩陣中的值為什麼都是負數？**  
A: 因為我們定義每一步的成本為 -1，所以從任何狀態到目標的累積獎勵都是負數。例如，需要 5 步才能到達目標的狀態值為 -5。

**Q: 應該使用哪個算法？**  
A: 
- 若要觀察隨機策略的性能，使用 **HW1-2 (Policy Evaluation)**
- 若要找到最優策略，使用 **HW1-3 (Value Iteration)**

**Q: 為什麼有些狀態有相同的值？**  
A: 如果多個狀態與目標的距離相同（在最優策略下），它們會有相同的值。

**Q: 障礙物如何影響值函數？**  
A: 障礙物會改變最短路徑，導致某些狀態的值變得更加負（因為需要更多步數才能繞開障礙物到達目標）。
